What happens when you initialize a repository? Why do you need to do it?

A .git directory is created. You need to do this because if you don't do it you won't
be able to track your files using git.

How is the staging area different from the working directory and the repository? What value do you think it offers?

The staging area is between the working directory and the repository, it is like a bridge. It makes you have more control with the
files you want to pass to the repository.

How can you use the staging area to make sure you have one commit per logical change?

Because I can see in the staging area the commits I want to send to the repository.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

They are helpful when you want to do something that doesn't affect the main branch. They help when you want to test new things.

How do the diagrams help you visualize the branch structure?

They are easier to understand.
